# 智能客户支持 Agent 平台部署与交付说明

## 1. 部署目标

```text
让客户侧 AI 帮助中心、客服工单工作台、知识库管理后台和数据看板可以通过线上地址访问。
```

第一版采用真实 MVP 上线方案：

```text
一个 Node Web Service 同时承载前端静态页面和后端 API；
SQLite 数据库写入持久磁盘；
大模型 Key 和模型配置通过服务端环境变量管理。
```

该方案用于验证业务闭环和系统交付路径，不等同于中大型企业最终生产架构。

## 2. 部署平台

```text
推荐平台：Render
部署方式：Web Service + Persistent Disk
配置文件：render.yaml
```

选择该方式的原因：

```text
1. 当前项目是 React + Express 单体服务，适合单服务部署
2. 后端需要调用大模型 API，必须运行在服务端
3. 当前数据库为 SQLite，需要持久磁盘保存数据
4. Render Web Service 可以配置构建命令、启动命令、环境变量和磁盘挂载
```

## 3. 部署前准备

需要准备：

```text
1. GitHub 仓库
2. Render 账号
3. 火山方舟 ARK_API_KEY
4. Seed 2.0 Pro 对应的 ARK_MODEL 或推理接入点 ID
```

不能提交到仓库的内容：

```text
1. .env
2. 真实 ARK_API_KEY
3. 任何真实密钥或平台账号密码
```

## 4. Render 环境变量

| 变量名 | 是否必填 | 说明 |
| --- | --- | --- |
| NODE_VERSION | 是 | 建议使用 24.14.1，保证 node:sqlite 可用 |
| DATABASE_DIR | 是 | Render 持久磁盘挂载目录，第一版使用 /var/data |
| ARK_BASE_URL | 是 | 火山方舟 OpenAI 兼容接口地址 |
| ARK_API_KEY | 是 | 火山方舟 API Key，只能配置在 Render 环境变量中 |
| ARK_MODEL | 是 | Seed 2.0 Pro 模型 ID 或推理接入点 ID |

Render 环境变量示例：

```text
NODE_VERSION=24.14.1
DATABASE_DIR=/var/data
ARK_BASE_URL=https://ark.cn-beijing.volces.com/api/v3
ARK_API_KEY=在 Render 控制台中填写
ARK_MODEL=在 Render 控制台中填写
```

## 5. 构建与启动命令

```bash
npm ci && npm run build
npm start
```

说明：

```text
npm run build 会生成前端 dist 文件；
npm start 会启动 Express 服务；
Express 服务会同时提供 API 和前端静态页面。
```

## 6. 数据持久化

本地开发默认数据库路径：

```text
data/support-agent.db
```

线上部署数据库路径：

```text
/var/data/support-agent.db
```

说明：

```text
DATABASE_DIR 指向 Render 持久磁盘挂载目录。
如果没有配置持久磁盘，SQLite 数据可能在服务重启或重新部署后丢失。
```

## 7. 线上访问路径

部署完成后，将 Render 生成的域名替换为实际地址：

| 端 | 路径 | 用途 |
| --- | --- | --- |
| 客户侧 AI 帮助中心 | / | 客户提问、AI 回答、提交工单 |
| 客服工单工作台 | /workspace | 客服查看工单、生成草稿、回复客户 |
| 知识库管理后台 | /knowledge | 运营维护知识内容 |
| 数据看板 | /dashboard | 查看 AI 效果和服务指标 |
| 健康检查 | /api/health | 检查后端服务是否正常 |

## 8. 上线验收清单

```text
1. 客户侧页面可以打开
2. 客户可以提交问题并获得 AI 回答
3. 回答可以显示引用来源
4. 客户可以反馈已解决或仍需帮助
5. 客户可以提交工单
6. 客服工作台可以看到工单
7. 客服可以生成回复草稿并回复客户
8. 客户可以看到客服回复并继续补充说明
9. 知识库后台可以新增、编辑、启用和停用知识
10. 数据看板可以显示咨询量、反馈、工单和知识统计
11. 服务重启后历史工单和知识仍然存在
12. 页面和文档中不暴露真实 API Key
```

## 9. 当前能力边界

```text
1. 当前使用 SQLite，适合 MVP 试点和小规模验证
2. 当前没有登录权限，后台地址知道后即可访问
3. 当前没有企业多租户隔离
4. 当前没有日志监控、告警和自动备份
5. 当前没有灰度发布和回滚流程
```

## 10. 生产化升级路径

如果进入正式生产环境，建议升级为：

```text
1. 前端独立部署到 CDN 或前端托管平台
2. 后端独立部署到云服务或容器平台
3. SQLite 替换为 PostgreSQL、MySQL 或托管数据库
4. 增加登录、角色权限和企业数据隔离
5. 增加日志、监控、告警和错误追踪
6. 使用 Secret Manager 或云平台密钥管理
7. 增加数据库备份与恢复策略
8. 增加灰度发布和回滚机制
```

## 11. 交付口径

```text
当前版本已经完成 AI 客服业务闭环和第一版线上部署准备。
第一版部署用于验证客户提问、知识库检索、AI 回答、工单流转、客服回复、知识运营和数据评估。
系统保留生产化升级路径，后续可围绕权限、数据库、监控、RAG 检索和 Prompt 管理继续迭代。
```
