# AI 产品经理学习路线

## 1. 你要在这个项目中学会什么

```text
业务层：
理解客户支持业务流程、客服效率指标、工单流转逻辑。

产品层：
能写需求文档、用户流程、功能优先级、指标体系和简历项目描述。

AI 层：
理解大模型、Prompt、RAG、Embedding、知识库、Agent、幻觉、置信度和人工兜底。

工程协作层：
理解前端、后端、数据库、API、部署、日志和线上可用性。

面试表达层：
能解释为什么选这个方向、解决什么问题、AI 怎么工作、如何评估效果、有什么风险和优化方向。
```

## 2. 第一阶段：先把项目讲清楚

你需要掌握：

- 什么是 B 端 AI 产品
- 什么是企业知识库
- 为什么客服场景适合 AI Agent
- AI 产品和普通产品的区别
- AI 产品为什么必须有人工兜底

你需要能回答：

```text
这个项目服务谁？
他们原来怎么解决问题？
痛点是什么？
为什么 AI 能解决？
为什么不是简单聊天机器人？
```

## 3. 第二阶段：掌握核心 AI 概念

你需要掌握：

- LLM：大语言模型
- Prompt：给模型的任务指令
- RAG：检索增强生成
- Embedding：把文本转成可检索的向量
- Agent：能按目标进行判断和执行的 AI 流程
- Hallucination：模型幻觉
- Human in the loop：人工确认和接管

你需要能回答：

```text
为什么不能直接把问题丢给大模型？
知识库检索在里面起什么作用？
AI 回答为什么要展示引用来源？
什么情况下应该转人工？
```

## 4. 第三阶段：完成项目设计

你需要产出：

- 需求文档
- 用户流程图
- 功能结构图
- 数据指标表
- 简历项目描述

## 5. 第四阶段：完成项目上线

你需要了解：

- 前端页面如何展示
- 后端接口如何工作
- 数据库保存什么
- 大模型 API 如何接入
- 项目如何部署到线上

## 6. 第五阶段：准备面试讲法

你需要准备：

```text
1 分钟项目介绍
3 分钟完整项目介绍
项目难点
AI 能力边界
指标设计
后续优化方向
```
