# 智能客户支持 Agent 平台需求文档

## 1. 项目背景

许多 B2B SaaS 企业的客户支持团队需要处理大量重复问题，例如账号配置、功能使用、计费规则、权限问题和故障排查。传统客服依赖人工查文档、人工分派工单，响应慢、成本高、知识沉淀差。

本项目通过企业知识库和 AI Agent，将客户问题自动检索、回答、分类、生成工单并进入处理流程，帮助企业提升支持效率。

## 2. 项目目标

```text
用户目标：
客户可以快速获得准确回答，复杂问题可以顺利转为工单。

企业目标：
降低重复咨询成本，提高问题解决效率，沉淀知识库和服务数据。

简历目标：
展示 AI 产品经理对业务场景、AI 能力边界、RAG、Agent 工作流、指标体系和产品闭环的理解。
```

## 3. 目标用户

```text
外部用户：B2B SaaS 企业客户、客户侧管理员、普通使用者
内部用户：客服人员、技术支持、客户成功经理、支持团队主管
```

## 4. 核心功能

### 4.1 用户端 AI 问答

用户输入问题后，系统基于企业知识库检索相关内容，并生成回答。

必须包含：

- 问题输入
- AI 回答
- 引用来源
- 置信度提示
- 已解决 / 仍需帮助反馈
- 未解决时生成工单

### 4.2 知识库管理

管理员维护企业文档和 FAQ。

必须包含：

- 文档列表
- 文档分类
- 文档内容查看
- 新增、编辑、删除知识内容

### 4.3 工单管理

AI 无法完整解决问题时，自动生成工单。

必须包含：

- 工单标题
- 问题描述
- AI 分类
- 优先级
- 负责部门
- 状态流转
- 客户反馈同步

### 4.4 数据看板

展示 AI 客服效果。

必须包含：

- 总咨询量
- AI 解决率
- 转人工率
- 知识命中率
- 工单数量

## 5. AI 能力边界

第一版不训练模型，重点使用大模型 API 和知识库检索完成业务闭环。

```text
AI 负责：
1. 理解用户问题
2. 从知识库中找到相关内容
3. 基于知识库生成回答
4. 判断是否需要转工单
5. 给工单打分类、优先级和负责部门

AI 不负责：
1. 无依据编造答案
2. 直接执行高风险操作
3. 替代人工处理所有复杂问题
```

## 6. 第一版范围

第一版目标是做出完整可演示闭环。

```text
包含：
用户端问答
管理端知识库
管理端工单
数据看板
基础 RAG 检索
可线上部署

暂不包含：
复杂权限体系
多租户企业隔离
真实客服系统集成
真实企业微信/飞书集成
模型微调
```
